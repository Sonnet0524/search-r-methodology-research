# SEARCH-R方法论

> Research Agent的核心工作方法论

**版本**: v1.0 | **更新**: 2026-03-07

---

## 📖 方法论概述

SEARCH-R是一套专为研究型Agent设计的方法论，指导Agent系统化地进行研究工作。

### 命名由来

**Search（搜索）+ Reflect（反思）= 持续探索真理**

研究本质是不断搜索真理的过程，每次循环都带来新的理解。

---

## 🔄 核心内容

### 1. SEARCH-R循环

完整的七阶段研究循环：

```
S - Survey（观察调研）
E - Explore（探索检索）
A - Analyze（分析思考）
R - Review（评审探讨）
C - Confirm（确认验证）
H - Harvest（收获产出）
R - Reflect（反思迭代）

循环：S → E → A → R → C → H → R → (回到S)
```

📖 [详细说明](search-r-cycle.md)

---

### 2. 研究深度标准

从第一性原理到实现思路的五层深度：

```
Level 0: 第一性原理
Level 1: 理论框架
Level 2: 设计原则
Level 3: 实现思路
Level 4: 具体实现 ❌（非Research Agent职责）
```

**目标深度**: Level 0-2

📖 [详细说明](research-depth.md)

---

### 3. Human角色定义

Human在研究过程中的双重角色：

```
角色一：信息传递者（不算"Human介入"）
- 传递研究背景
- 提供研究资源
- 不影响研究进程

角色二：关键决策者（算"Human介入"）
- 研究方向选择
- 理论框架验证
- 决定是否采纳
```

**原则**: Human参与最小化

📖 [详细说明](human-role.md)

## 🎯 方法论特点

### 特点1: 系统性

- ✅ 完整的研究循环
- ✅ 明确的阶段划分
- ✅ 标准化的产出

### 特点2: 深度性

- ✅ 追求第一性原理
- ✅ 建立理论框架
- ✅ 明确设计原则

### 特点3: 可追溯性

- ✅ 记录研究过程
- ✅ 记录决策过程
- ✅ 形成知识沉淀

### 特点4: 迭代性

- ✅ 持续优化方法论
- ✅ 反思研究过程
- ✅ 改进研究方法

## 📊 使用指南

### 快速开始

1. **理解方法论** - 学习SEARCH-R循环
2. **明确深度目标** - 确定研究深度要求
3. **定义Human角色** - 明确何时需要Human参与
4. **开始研究循环** - 从Survey（S）开始

### 适用场景

**适合**:
- ✅ 需要系统性研究的复杂问题
- ✅ 需要从第一性原理思考的设计
- ✅ 需要记录研究过程以便复盘
- ✅ Agent与Human深度协作的研究

**不适合**:
- ❌ 简单的问答式查询
- ❌ 纯执行层面的工作
- ❌ 不需要深度思考的任务

## 🔄 方法论应用

### 完整研究循环

```
Step 1: Survey（观察调研）
- 观察实践过程
- 发现问题和模式
- 记录观察笔记

Step 2: Explore（探索检索）
- 检索相关知识
- 调研现有研究
- 整理知识图谱

Step 3: Analyze（分析思考）
- 构建理论框架
- 提出假设
- 逻辑推演

Step 4: Review（评审探讨）
- Human参与评审
- 深度讨论
- 方向调整

Step 5: Confirm（确认验证）
- 实践验证
- 收集数据
- 分析结果

Step 6: Harvest（收获产出）
- 沉淀研究成果
- 编写最佳实践
- 建立知识库

Step 7: Reflect（反思迭代）
- 反思研究过程
- 优化方法论
- 准备下一轮
```

### 研究深度控制

```
每个研究阶段都应该评估：

Level 0: 是否理解了"为什么"？
Level 1: 是否建立了理论模型？
Level 2: 是否明确了设计原则？

目标：达到Level 0-2的深度理解
```

### Human参与控制

```
每个决策点都应该判断：

问题类型：
- 信息传递？→ 不需要Human介入
- 关键决策？→ 需要Human介入

介入原则：
- 能自主判断就不呼叫
- 积累问题批量处理
- 充分准备后再介入
```

## 🎓 最佳实践

### 实践1: 持续追问

```
陈述 → 追问"为什么" → 更深层理解 → Level 0达成
```

### 实践2: 建模思维

```
观察 → 提取模式 → 建立模型 → 形式化表达 → Level 1达成
```

### 实践3: 原则提取

```
现象 → 分析共性 → 提取原则 → 明确优先级 → Level 2达成
```

### 实践4: 批量介入

```
收集问题 → 整理背景 → 批量讨论 → 一次性解决
```
