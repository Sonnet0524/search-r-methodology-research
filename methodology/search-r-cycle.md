# SEARCH-R方法论详解

> 研究型Agent的标准工作循环

**版本**: v1.0 | **更新**: 2026-03-07

---

## 📖 方法论概述

SEARCH-R是一个**研究循环方法论**，指导研究型Agent系统化地进行研究工作。

### 命名由来

**Search（搜索）+ Reflect（反思）= 持续探索真理**

研究本质是不断搜索真理的过程，每次循环都带来新的理解。

---

## 🔄 SEARCH-R循环

```
S - Survey（观察调研）
E - Explore（探索检索）
A - Analyze（分析思考）
R - Review（评审探讨）
C - Confirm（确认验证）
H - Harvest（收获产出）
R - Reflect（反思迭代）

循环：S → E → A → R → C → H → R → (回到S)
```

---

## 📋 各阶段详解

### S - Survey（观察调研）

**目标**：从实践中发现问题和模式

**输入**：
- 项目背景和上下文
- 实践观察资料
- 现有问题和痛点

**活动**：
- 观察实践过程
- 识别关键现象
- 发现潜在问题
- 记录观察笔记

**输出**：
- 观察笔记（observation/）
- 问题清单
- 待探索的方向

**质量门控**：
- 是否发现有意义的问题？
- 问题是否值得研究？
- 是否需要Human确认研究方向？

**示例**：
```
观察场景：Agent Team协作
发现问题：
- Agent间信息传递效率低
- Context占用过多
- 协作冲突频繁

产出：observation/agent-collaboration-issues.md
```

---

### E - Explore（探索检索）

**目标**：检索相关知识和现有研究

**输入**：
- 观察阶段发现的问题
- 检索关键词
- 知识源列表

**活动**：
- 文献检索
- 技术调研
- 案例分析
- 知识整理

**输出**：
- 检索报告（retrieval/）
- 知识图谱
- 关键概念清单

**质量门控**：
- 是否找到足够的相关知识？
- 检索结果是否相关？
- 是否需要Human提供更多资源？

**示例**：
```
检索方向：Agent协作模式
检索内容：
- Multi-Agent System研究
- Agent Communication Language
- 协作框架对比

产出：retrieval/agent-collaboration-research.md
```

---

### A - Analyze（分析思考）

**目标**：深度理论构建和分析

**输入**：
- 观察笔记
- 检索报告
- 现有理论

**活动**：
- 理论构建
- 模式识别
- 假设提出
- 逻辑推演

**输出**：
- 理论文档（theory/）
- 概念模型
- 设计原则

**质量门控**：
- 理论是否逻辑自洽？
- 是否达到Level 0-2深度？
- 是否需要Human评审理论？

**示例**：
```
理论构建：文档化交互模式
核心概念：
- Agent通过文档异步交互
- 分层文档体系
- Context最小化原则

产出：theory/document-based-interaction.md
```

---

### R - Review（评审探讨）

**目标**：Human参与评审和探讨

**输入**：
- 理论文档
- 设计方案
- 待确认问题

**活动**：
- Human评审理论
- 深入探讨
- 方向调整
- 决策确认

**输出**：
- 评审记录
- 决策文档
- 改进建议

**质量门控**：
- Human是否认可理论？
- 是否需要调整研究方向？
- 是否有新的洞察？

**示例**：
```
评审内容：质量门控理论
Human反馈：
- 理论方向正确
- 需要明确量化标准
- 考虑实践可行性

产出：discussion/quality-gate-review.md
```

---

### C - Confirm（确认验证）

**目标**：在实践中验证理论

**输入**：
- 理论文档
- 验证方案
- 实践环境

**活动**：
- 设计验证实验
- 实施验证
- 收集数据
- 分析结果

**输出**：
- 验证报告
- 实践数据
- 改进点

**质量门控**：
- 验证是否充分？
- 数据是否支持理论？
- 是否需要更多验证？

**示例**：
```
验证内容：文档化交互效果
验证方法：
- 对比传统方式
- 测量Context占用
- 统计协作冲突

产出：validation/document-interaction-validation.md
```

---

### H - Harvest（收获产出）

**目标**：沉淀研究成果

**输入**：
- 理论文档
- 验证报告
- 实践经验

**活动**：
- 整理研究成果
- 编写最佳实践
- 制作使用指南
- 建立知识库

**输出**：
- 研究报告
- 最佳实践文档
- 使用指南
- 知识库更新

**质量门控**：
- 产出是否完整？
- 是否可复用？
- 是否需要Human确认？

**示例**：
```
成果沉淀：Agent协作框架
产出文档：
- README.md
- 最佳实践指南
- 模板和工具
- 知识库索引

产出：harvest/agent-team-framework-release.md
```

---

### R - Reflect（反思迭代）

**目标**：持续优化研究方法

**输入**：
- 整个研究过程
- 成果和问题
- Human反馈

**活动**：
- 反思研究过程
- 识别改进点
- 优化方法论
- 准备下一轮

**输出**：
- 反思笔记（reflection/）
- 方法论改进
- 下一轮研究计划

**质量门控**：
- 反思是否充分？
- 是否发现方法论问题？
- 是否需要重大调整？

**示例**：
```
反思内容：SEARCH-R方法论实践
发现改进：
- Review阶段需要更明确标准
- Confirm阶段需要更多验证
- Harvest阶段需要模板化

产出：reflection/search-r-methodology-improvement.md
```

## 🔄 循环特点

### 1. 迭代性

- 不是一次性完成
- 每轮循环加深理解
- 持续改进成果

### 2. 灵活性

- 不必严格按顺序
- 可以跳过某些阶段
- 可以重复某些阶段

### 3. 适应性

- 根据课题调整
- 根据进展调整
- 根据Human反馈调整

## 📊 使用场景

### 完整循环

**适用场景**：全新研究课题

**流程**：S → E → A → R → C → H → R → (循环)

**时间**：多个会话

### 快速循环

**适用场景**：已有研究基础

**流程**：A → R → H → R

**时间**：1-2个会话

### 单点深入

**适用场景**：需要深入研究某个方面

**流程**：在某个阶段停留，深入探索

**时间**：根据需要

## 🎯 质量标准

### 每个阶段的质量要求

| 阶段 | 质量标准 | 评估方法 |
|------|---------|---------|
| Survey | 发现真实问题 | Human确认 |
| Explore | 检索充分全面 | 覆盖率评估 |
| Analyze | 理论逻辑自洽 | 同行评审 |
| Review | Human充分参与 | 反馈质量 |
| Confirm | 验证数据充分 | 数据分析 |
| Harvest | 产出可复用 | 实践检验 |
| Reflect | 反思深入透彻 | 改进效果 |
