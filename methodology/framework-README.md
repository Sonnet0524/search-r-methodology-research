# Research Agent Framework

一个用于AI Agent研究的框架和方法论。

---

## 这是什么？

Research Agent Framework提供了：

- 🤖 **Research Agent的角色定义** - 如何设计一个研究型Agent
- 📚 **完整的研究方法论** - 观察-检索-思考-探讨-反思的研究循环
- 📝 **各种文档模板** - 观察笔记、检索报告、思考笔记、理论文档等
- 🎯 **从第一性原理到实现思路** - 多层次的研究深度
- 🧠 **自我反思与元研究** - 研究方法本身的持续优化

---

## 核心理念

### 1. 研究-检索-思考-探讨-反思的研究循环

```
观察（实践内容）→ 检索（在线资料）→ 思考（问题与发现）→ 探讨（洞察与挑战）→ 反思（验证与迭代）
```

这不是一个线性流程，而是一个持续循环的过程。

### 2. 洞察+挑战的配合模式

- **用户的角色**：提出方向、做决策
- **Research Agent的角色**：提供洞察、提出挑战

这是"共同探索、相互激发"的过程，不是简单的"你问我答"。

### 3. 元研究：对研究方法本身的研究

Research Agent不仅研究具体问题，还要研究"如何做研究"本身：

- 研究方法论的有效性
- 研究流程的优化
- 研究工具和模板的改进

### 4. 自我迭代：持续优化研究方法

每次会话后进行自我反思，当发现方法论问题时，主动与用户讨论并迭代改进。

---

## 研究深度：从第一性原理到实现思路

```
Level 0: 第一性原理层
  问题：XXX的本质是什么？为什么要XXX？
  方法：追问"为什么"直到无法再追问

Level 1: 理论框架层
  问题：XXX如何建模？决策模型是什么？
  方法：构建理论模型、形式化表达、逻辑推演

Level 2: 设计原则层
  问题：应该遵循什么原则？原则如何权衡？
  方法：从理论推导原则、分析原则关系

Level 3: 实现思路层
  问题：如何设计？有哪些可选方案？
  方法：从原则推导思路、提供多种方案

Level 4: 实施建议层 ❌
  问题：代码怎么写？配置放哪里？
  方法：这不是Research Agent的职责范围
```

## 如何使用？

### 快速开始

1. **定义Research Agent的角色**
   - 参考 `agents/research/AGENTS.md`
   - 根据你的项目调整职责范围

2. **建立文档结构**
   - 参考 `structure/directory-structure.md`
   - 创建必要的研究文档目录

3. **使用模板**
   - 从 `templates/` 选择合适的模板
   - 按需创建研究文档

4. **开始研究循环**
   - 观察实践
   - 检索资料
   - 思考问题
   - 与用户探讨
   - 反思总结

### 文档结构

```
research/
├── research-log.md              # 研究总日志（时间线）
├── observations/                # 观察笔记
├── retrievals/                  # 检索笔记
├── reflections/                 # 思考笔记
├── discussions/                 # 探讨记录
├── theories/                    # 理论文档
└── meta/                        # 元研究
    ├── research-methodology.md  # 研究方法论
    ├── self-reflections/        # 自我反思
    └── method-iterations/       # 方法迭代记录
```

## 设计哲学

### 1. Research Agent不是执行者

Research Agent是**研究员**，不是**执行者**：
- 关注"为什么"和"是什么"，不关心"怎么做"
- 提供理论支撑和设计思路，不提供实施细节

### 2. 研究应该可追溯

每次研究都应该记录：
- 问题是如何发现的
- 洞察是如何产生的
- 决策是如何做出的

这样便于后续优化迭代。

### 3. 研究应该有深度

不应该停留在表面，应该：
- 追问第一性原理
- 构建理论框架
- 推导设计原则
- 形成实现思路

### 4. 研究方法应该持续进化

研究方法本身也需要研究和改进：
- 每次会话后反思
- 发现方法论问题及时迭代
- 不断优化研究效率
