# 介入类型详解

## 介入类型概述

质量门控机制定义了四种Human介入类型，每种类型有不同的触发条件和行动要求。

---

## NONE - 无需介入

### 触发条件
- 确定性 = HIGH
- 可接受性 = HIGH
- 无认知混淆

### 含义
研究成果质量高，可以继续推进

### Agent行动
- 继续研究
- 记录成果
- 进入下一阶段

### 示例
```
研究成果：发现了Agent协作的核心模式
确定性：HIGH（逻辑清晰，有多个案例支撑）
可接受性：HIGH（符合研究预期）
混淆：无
→ 结论：继续研究
```

---

## CLARIFICATION - 请求澄清

### 触发条件
- 存在认知混淆
- 理解歧义
- 概念不清

### 含义
Agent对某些内容理解不确定，需要澄清

### Agent行动
- 明确列出混淆点
- 提供上下文
- 请求Human澄清

### 示例
```
混淆点："Agent的生命周期"指什么？
- 是指进程生命周期？
- 还是指任务生命周期？
- 还是指会话生命周期？

→ 请求Human澄清定义
```

---

## DIRECTION - 请求方向指导

### 触发条件
- 确定性 = MEDIUM
- 可接受性 = MEDIUM
- 研究方向需要确认

### 含义
研究基本正确，但需要Human确认方向

### Agent行动
- 汇报当前进展
- 列出可能方向
- 请求Human选择

### 示例
```
当前进展：发现了三种Agent协作模式
- 模式A：中心化协调
- 模式B：去中心化协作
- 模式C：混合模式

问题：应该深入研究哪种模式？

→ 请求Human指定研究方向
```

---

## DECISION - 请求决策

### 触发条件
- 确定性 = LOW
- 可接受性 = LOW
- 需要重大决策

### 含义
研究成果有重大问题，需要Human做出决策

### Agent行动
- 说明问题所在
- 分析可能原因
- 提供选项
- 请求Human决策

### 示例
```
问题：研究结果与预期完全不符

发现：
- 假设A被数据反驳
- 假设B也证据不足

选项：
1. 重新设计研究方案
2. 调整研究目标
3. 终止该研究方向

→ 请求Human做出决策
```

---

## 介入类型决策树

```
研究成果
    │
    ├─ 存在混淆？ ────→ CLARIFICATION
    │
    ├─ 确定性=LOW？ ──→ DECISION
    │
    ├─ 可接受性=LOW？ → DECISION
    │
    ├─ 确定性=MEDIUM？ → DIRECTION
    │
    └─ 确定性=HIGH ∧ 可接受性=HIGH → NONE
```
