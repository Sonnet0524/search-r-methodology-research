# 理论框架模式

## 什么是理论框架？

理论框架是对研究问题及其关系的系统性解释，包含核心概念、命题和边界条件。

## 框架类型

### 概念框架
描述概念及其关系

```
概念A ──关系1──→ 概念B
  │              │
关系2          关系3
  ↓              ↓
概念C ←─关系4─── 概念D
```

### 过程框架
描述动态过程

```
输入 → 处理1 → 处理2 → 输出
         ↓         ↓
      反馈1    反馈2
```

### 分层框架
描述层次结构

```
Layer 0: 第一性原理
    ↓
Layer 1: 设计原则
    ↓
Layer 2: 实现思路
    ↓
Layer 3: 具体实现
```

## 构建步骤

```
1. 识别核心概念
   ↓
2. 定义概念关系
   ↓
3. 构建命题陈述
   ↓
4. 明确边界条件
   ↓
5. 验证框架一致性
```

```markdown
| 概念 | 定义 | 来源 |
|------|------|------|
| 概念A | ... | ... |

## 二、概念关系

| 关系 | 类型 | 说明 |
|------|------|------|
| A→B | 因果 | ... |

## 三、核心命题

1. 命题1: ...
   - 依据: ...
   - 推导: ...

## 四、边界条件

- 适用范围: ...
- 不适用场景: ...
- 前提假设: ...

## 五、验证方案

- 指标: ...
- 方法: ...
```

## 常见框架模式

| 模式 | 特点 | 适用场景 |
|------|------|---------|
| 输入-处理-输出 | 线性流程 | 系统分析 |
| 层次结构 | 分层抽象 | 架构设计 |
| 反馈循环 | 迭代优化 | 持续改进 |
| 因果链 | 因果关系 | 问题分析 |
